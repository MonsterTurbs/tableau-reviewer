🌊Foundation

Big Data Terms

😊Simple Explanation

Raw data is like crude oil: valuable, but not immediately useful. It must be cleaned, modeled, analyzed, and presented clearly.

📌Key Terms

Big Data has high volume, high velocity, and high variety. ETL means Extract, Transform, Load.

🧪Daily Life Example

An online store produces order records, clicks, product views, customer locations, and payments. A dashboard turns those events into a story.

📊Structured Data

Tables, rows, and columns. Example: a sales table with Order ID, Date, Product, Quantity, and Sales.

🧩Semi-Structured Data

XML, JSON, and server logs. It has organization, but it does not always fit neatly into rows and columns.

🎧Unstructured Data

Videos, images, audio, and free text. It needs extra processing before it becomes easy to analyze.

💡 Data becomes valuable only when converted into insight.
✅Takeaway

Data becomes valuable when it becomes insight that helps someone make a better decision.

Business Intelligence

💡Core Idea

Business Intelligence, or BI, is the bridge between raw data and business decisions.

BI in Simple Terms

BI is like a translator. Raw data speaks in rows, columns, and files. BI turns that into charts and dashboards that managers can use to decide.

Simple example: Instead of reading 20,000 sales rows, a manager sees one dashboard showing sales, profit, top products, and warning areas.

👔 Decision MakersManagers, finance, HR, operations, and leaders who need clear answers before choosing an action.
🧹 Data AnalystCleans, connects, aggregates, and checks data so business questions can be answered correctly.
🧠 Data ScientistUses mining, machine learning, and models to predict outcomes or discover unknown insights.
🌉 BIThe bridge between raw data and business decisions.
😊Car Dashboard Analogy

Managers do not need raw engine signals. Like a driver, they need speed, fuel, warning lights, and direction.

🧪Shop Example

Instead of showing thousands of sales rows, a BI dashboard can show total sales, profit, monthly trend, top products, and city filters.

✅Takeaway

BI turns data into understandable information so people can decide faster and smarter.

Data Visualization

💡Purpose

Data visualization converts numbers into charts, maps, and visual patterns that people can understand quickly.

From Numbers to a Story

Data visualization means using graphical elements like bars, lines, maps, colors, and labels to show data clearly.

Simple example: A table says sales were 10K, 12K, 15K, then 9K. A line chart instantly shows the rise and sudden drop.

📊 Data VisualizationUsing graphical elements to show data so people can understand it faster.
📖 StorytellingBuilding a clear narrative from data: what happened, why it matters, and what action comes next.
🔁 PatternA repeated or meaningful behavior in the data, such as every December having higher sales.
📈 TrendDirection over time, such as sales going up, going down, or staying flat.
🧪Line Chart Example

A monthly sales table can hide a sudden drop. A line chart makes the drop visible immediately.

🧪Bar Chart Example

If you need to compare product categories, a sorted bar chart usually beats a decorative chart.

✅Takeaway

A good chart does not just look nice. It makes the answer obvious.

Excel vs BI Tools

⚖️Comparison

Excel is great for small manual work. BI tools are better when reporting must be automated, secure, centralized, refreshed, and interactive.

Top 3 BI Tools

Common BI tools in the market are Tableau, Power BI, and Qlik. All can create business dashboards and visual analysis, but each has different strengths.

Simple example: They are like different cars. All can take you to the destination, but the controls, ecosystem, cost, and driving feel are different.

TableauStrong in visual analytics and interactive exploration.
Power BIStrong in the Microsoft ecosystem and enterprise adoption.
QlikKnown for associative data modeling and exploration.
🤖 AutomationAutomatic refresh or update of data and reports.
🏋️ CapacityWhether the tool can handle large data volume and many users.
🔐 SecurityControl over who can see data, dashboards, and reports.
🧾 Row-Level SecurityEach user only sees the rows allowed for their role.
✅ Single Source of TruthOne trusted report or dashboard instead of many conflicting spreadsheet versions.
⚠️Spreadsheet Trap

Files named final, final_v2, and final_REAL create confusion. A Tableau dashboard can become the single trusted version.

🧪Team Example

Instead of emailing a new spreadsheet every morning, publish one dashboard and refresh it on a schedule.

✅Takeaway

Use Excel for quick analysis; use Tableau for scalable business reporting.

🏗️Tableau Ecosystem

Tableau Architecture

💡Live vs Extract

A live connection queries the source directly. An extract stores an optimized copy for faster performance and scheduled refresh.

Freshness vs Speed

This is one of the most practical Tableau decisions. If you choose poorly, the dashboard can become slow or the numbers can become outdated.

Simple rule: Live favors fresh data. Extract favors speed, stability, and portability.

Architecture in Plain English

Tableau Desktop moves from data source, to worksheet, to dashboard, to story. Tableau Server or Cloud lets many users securely view, refresh, and interact with dashboards.

Server example: A user opens a dashboard, Tableau checks login and permissions, runs the needed queries, renders the view, and sends it back to the browser.

🏢 On-PremisesYour organization manages hardware, software, security, updates, and operations.
☁️ IaaSA cloud provider owns the hardware, but your team still manages Tableau Server software.
🌐 SaaS / Tableau CloudThe provider manages infrastructure and platform maintenance.
🧱 Key Server PartsGateway, Application Server, VizQL, Data Server, Repository, File Store, Cache Server, and Backgrounder work together.
📌File Types

.twb is a workbook. .twbx is a packaged workbook. .tds is a data source. .tdsx is a packaged data source. .hyper is an extract.

🧩Server Components

Gateway, Application Server, VizQL Server, Data Server, Repository, File Store, Cache Server, and Backgrounder work together to serve dashboards.

🧠Memory Hook

Live = asks the source. Extract = Tableau has its own optimized copy.

🔗Data Modeling

Combining Data

💡Foundation

Good data modeling prevents wrong numbers. A beautiful dashboard with wrong data is still dangerous.

Data Modeling in Simple Terms

Data modeling is organizing tables and defining how they relate. It is the backbone of the dashboard because it decides how rows connect and how totals are calculated.

Simple example: Orders connect to Customers by Customer ID. If that relationship is wrong, sales by customer may duplicate or disappear.

📌Core Methods

Relationships connect logical tables flexibly. Joins merge columns side by side. Unions stack rows. Data blending combines different data sources at worksheet level.

⚠️Join Risk

Wrong joins can duplicate or remove rows, which can make sales, profit, counts, and averages incorrect.

🧠Memory Hook

Join adds columns. Union adds rows. Relationship keeps tables separate until Tableau queries them.

  • Use union for similar monthly or yearly files.
  • Use relationships for different entities in one model.
  • Use joins when you truly need physical merging.
  • Use blending when sources cannot be combined normally.
🏷️Data Modeling

Tableau Metadata

💡Metadata Meaning

Metadata describes fields: name, data type, geographic role, default aggregation, and whether a field is a dimension or measure.

😊Dimensions and Measures

Dimensions answer 'by what?' Measures answer 'how much?' Example: Sales by Category uses Category as dimension and Sales as measure.

⚠️Data Type Warning

Keys used in relationships or joins must have matching data types. Customer ID as text will not cleanly match Customer ID as number.

🧠Memory Hook

Discrete fields create headers or categories. Continuous fields create axes or ranges.

✏️Data Modeling

Renaming and Aliases

💡Why It Matters

Clear names make a workbook easier to understand, maintain, and hand off to another person.

🌟Best Practice

Use business-friendly names. Avoid names like Calc 1 when a clear name is possible.

🧪Example

Rename Cust_ID to Customer ID. Use aliases to show DE as Germany or US as United States.

✅Takeaway

Rename fields for readability. Use aliases for values inside a field.

Tableau Parameters

💡Simple Explanation

A parameter is like a dashboard knob. By itself it does nothing; it must be connected to a calculation, filter, reference line, title, or action.

🧪Threshold Example

Instead of hardcoding Profit < 10000 as red, create a Profit Threshold parameter so the user can choose the threshold.

🧪Measure Switch Example

Use a parameter to let users switch the chart between Sales, Profit, and Quantity.

✅Takeaway

Parameters make one view flexible enough to answer multiple requirements.

⚡Interactivity

Tableau Actions

💡Purpose

Actions make dashboards interactive. A user can click, hover, or select something and trigger a change.

📌Action Types

Filter action, highlight action, URL action, go-to-sheet action, set action, and parameter action.

🧪Example

Click a product category in a bar chart, and the sales trend chart filters to that category.

✅Takeaway

Actions turn static reports into guided exploration tools.

🧮Calculations

Tableau Calculations

💡Purpose

Calculations create custom fields for business logic, KPIs, labels, transformations, and advanced charts.

📌Calculation Types

Row-level calculations happen per row. Aggregate calculations happen after aggregation. Table calculations use the visible table. LOD expressions control the calculation level.

🧪Example

Profit Ratio = SUM([Profit]) / SUM([Sales]).

⚠️Common Mistake

Many calculation errors are not syntax errors. They happen because the calculation is being performed at the wrong level of detail.

  • Number functions: ROUND, CEILING, FLOOR, ABS.
  • String functions: LEFT, RIGHT, MID, SPLIT, TRIM.
  • Date functions: TODAY, DATEPART, DATETRUNC, DATEDIFF.
  • Null functions: ISNULL, IFNULL, ZN.
  • Logical functions: IF, CASE, AND, OR, NOT.
  • LOD expressions: FIXED, INCLUDE, EXCLUDE.
📈Visualization

Tableau Charts

💡Main Rule

Choose the chart based on the question, not because the chart looks fancy.

📊Chart Menu

Use bar charts for category comparison, line charts for trends, maps for geography, histograms or box plots for distribution, and scatter plots for relationships between two measures.

🧪Scenario Example

Question: Which category sells the most? Use a sorted bar chart. Question: Is profit rising over time? Use a line chart.

✅Takeaway

A simple chart that answers the question is better than a complex chart that confuses the user.

🧱Visualization

Tableau Dashboards

💡Dashboard Meaning

A dashboard is not a pile of charts. It is a designed decision-making interface.

📌Layout Terms

Tiled layout, floating layout, horizontal container, vertical container, padding, filters, legends, and dashboard actions.

🌟Best Practice

Start with the purpose and user question. Keep spacing consistent. Put KPIs where users can see them quickly. Test the dashboard in viewer mode.

🧪Example

A sales dashboard may have KPI cards at the top, a monthly trend in the middle, product comparisons below, and filters on the side.

🚀Project

Tableau Project Workflow

💡Project Principle

Real Tableau work starts with requirements, not colors.

🔁Project Flow

Analyze requirements, choose charts, design a mockup, prepare the data source, build calculations, build charts, arrange containers, format, test, add interactivity, and deliver.

🧪Client Example

If the client asks for an executive dashboard, translate each requirement into a KPI, chart, filter, or navigation element.

✅Takeaway

Requirements drive chart choices, chart choices drive layout, and testing protects correctness.

💎Cheat Sheet

Final Cheat Sheet

💎Core Concepts

BI supports decisions. Visualization turns data into visuals. Tableau builds worksheets, dashboards, and stories. Dimensions group data. Measures calculate numbers.

🧠Remember

Relationship = flexible logical connection. Join = side-by-side merge. Union = row stacking. Parameter = user-controlled value. Action = interactive behavior. LOD = controlled calculation level.

⚠️Avoid These Mistakes

Do not publish private data to Tableau Public. Do not trust guessed data types without checking. Do not join tables without validating totals. Do not build dashboards before understanding requirements.

✅Takeaway

Good Tableau work combines data thinking, visual thinking, and project thinking.

Tableau Screen Examples in Simple Terms

These are the exact Tableau screenshots from the examples. Click any card to open a larger modal view, then use Next and Previous to study each chart or Tableau screen step by step.

Multi-Line Trend Chart by Genre

Charts / marks used: line marks, Year on Columns, SUM(Global Sales) on Rows, Genre on Color, Tooltip enabled.

This view uses a line chart to show how global sales changed across years for each game genre. Each colored line represents one genre, such as Sports, Action, Shooter, or Role-Playing.

In simple terms, this is like tracking several racers over time. When a line goes up, that genre increased in sales. When it goes down, that genre declined. The tooltip appears when the user hovers over a point and shows the exact Genre, Year, and Global Sales value.

This chart is best when the question is about trends, peaks, drops, or comparison across time. It is useful for seeing which genres became popular, which ones declined, and which year had the strongest sales.

Stacked Bar Small Multiples

Charts / marks used: stacked bar marks, Year bin on Columns, two measures on Rows, Genre on Color, filters for Platform and Year bin.

This screen shows two stacked bar charts placed one above the other. The top chart shows SUM(Global Sales), while the lower chart shows a second measure, likely Global Sales minus EU Sales.

In simple terms, each bar is a total for a time group, and each color inside the bar is a genre contribution. The taller the bar, the larger the sales. The colored sections explain what makes up the total.

This view is useful when you want to compare totals and composition at the same time. It also demonstrates small multiples, because Tableau is showing related charts in separate rows so the viewer can compare measures without mixing them in one chart.

Dual-Axis Bar and Line Chart

Charts / marks used: bar marks for SUM(Global Sales), line marks for SUM(NA Sales), Year on Columns, dual axis with Measure Names legend.

This view combines two chart types in one worksheet. The blue bars show Global Sales by year, while the orange line shows NA Sales by year. The left axis belongs to Global Sales, and the right axis belongs to NA Sales.

In simple terms, the bars show the big worldwide total, and the line shows how one region follows or differs from that total. If both rise and fall together, North America is following the global pattern. If they separate, the region may be behaving differently.

This chart is useful when two related measures need to be compared over time. It should be used carefully because dual-axis charts can confuse viewers if the axes are not clearly labeled.

Filled Map by State or Province

Charts / marks used: map marks, generated Longitude on Columns, generated Latitude on Rows, State/Province on Color.

This screen shows a filled map of the United States. Tableau generated Latitude and Longitude automatically, then filled each state shape with a different color based on the State/Province field.

In simple terms, Tableau recognized geographic data and drew a map. Each state is a mark. Color is being used to separate states or categories, not necessarily to show higher or lower value.

This view is useful when the analysis depends on location. It can be improved by coloring states by a meaningful measure such as Sales, Profit, Population, or Store Count instead of using many random category colors.

Packed Bubble Chart by Genre

Charts / marks used: circle marks, Genre on Label and Color, SUM(Global Sales) on Size, Show Me packed bubbles option.

This view uses circles to compare game genres by sales. Larger bubbles represent larger sales totals. For example, Action has the largest bubble, while Adventure and Strategy have smaller bubbles.

In simple terms, bigger circle means bigger number. The label inside each bubble tells the genre and the sales value. The color helps separate the bubbles visually.

Packed bubbles are good for a quick visual impression, but they are not the best chart for precise ranking. If exact comparison matters, a sorted bar chart is usually clearer.

Pie Chart by Genre

Charts / marks used: pie marks, Genre on Color and Label, SUM(Global Sales) on Angle and Label.

This view shows how total global sales are divided across game genres. Each slice represents one genre, and the size of the slice depends on the sales value.

In simple terms, the whole pie is total sales. Every slice is one genre share of that total. Action, Sports, and Shooter have larger slices because they contribute more sales.

This chart is useful for simple part-to-whole explanation. However, because there are many slices, it can become hard to compare categories precisely. A bar chart may be easier if the goal is ranking genres from highest to lowest.

Stacked Area Chart for NA Sales

Charts / marks used: area marks, Year on Columns, SUM(NA Sales) on Rows, Genre on Color and Label.

This view shows North America sales over time, with colored layers for different genres. The total height of the stacked area shows total NA Sales, while each colored band shows how much each genre contributes.

In simple terms, it is like a mountain made of colored layers. The mountain gets taller when total sales increase. The thickness of each layer shows how important that genre was during that year.

This chart is useful for showing volume over time and composition over time. It is especially good for seeing a boom period, but it can be harder to read exact values compared with a line or bar chart.

Data Source Inner Join Screen

Tableau feature used: physical join canvas, Inner Join selected, Employee ID join clause, data preview grid.

This screen is from the Tableau Data Source page. Two tables, Demographics and JobTitle, are joined together using Employee ID. The selected join type is Inner Join.

In simple terms, Tableau is matching rows from two tables where the Employee ID is the same. If an employee exists in both tables, the row appears in the result. If the ID exists in only one table, an inner join removes it from the joined result.

This screen is important because joins affect the numbers shown later in charts. A wrong join can duplicate rows, remove records, or create incorrect totals. Always inspect the preview grid and validate row counts after joining tables.

Workbook plus packaged data/assets

✅Dashboard Quality

Dashboard Design Checklist

Use this checklist before you publish or present a Tableau dashboard. A good dashboard should be easy to read, focused on decisions, and trustworthy.

📝
Clear title

The title should tell users what the dashboard is about, such as Sales Performance by Region.

🔢
KPI row

Place the most important numbers at the top so users can understand the status quickly.

🎛️
Useful filters

Add filters that help users answer real questions, like Date, Region, Segment, or Category.

🎨
Consistent colors

Use the same colors for the same meanings. Do not change color rules from chart to chart.

🏷️
Readable labels

Labels should be large enough, clear enough, and placed only where they help understanding.

🧹
No chart clutter

Remove extra gridlines, repeated labels, unnecessary legends, and decorations that do not help the question.

🧮
Tested calculations

Check totals, filters, calculated fields, and dashboard actions before sharing with others.

👀
Quick scan test

A viewer should understand the main message in a few seconds before exploring the details.
